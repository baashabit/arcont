# ARCONT Pro Redesign Strategy

## Objetivo

Transformar el sistema actual bajo la marca ARCONT en una plataforma profesional sin perder la lógica de negocio ya capturada.

## Tesis

El sistema actual no está vacío ni mal orientado. El problema principal es que:
- la experiencia es legacy,
- la navegación es acumulativa,
- la jerarquía visual es débil,
- la seguridad visible está por debajo de estándar moderno,
- y la percepción del producto no corresponde al valor real de sus módulos.

## Nueva estructura recomendada

### 1. Executive Suite
- dashboard de dirección
- absorción
- ventas cerradas
- pipeline consolidado
- backlog de gestoría
- estado de inventario
- estado de requisiciones y blackboard

### 2. Commercial Suite
- prospectos
- inventario de vivienda
- alta venta
- contado
- validación de venta
- gestoría
- titulación

### 3. Operations Suite
- blackboard
- cumplimiento
- tareas por proyecto
- seguimiento por responsable
- alertas y vencimientos

### 4. Backoffice Suite
- usuarios
- catálogos
- notaría
- unidad valuadora
- líneas de crédito
- requisiciones
- materiales
- reportes

## Principios de rediseño

- Diseñar por procesos, no por listados.
- Diseñar por rol, no por menú histórico.
- Reducir el peso de las tablas como experiencia principal.
- Dejar dashboards y vistas detalle como primera capa.
- Mantener compatibilidad gradual con el backend actual.

## Fases sugeridas

### Fase 1
- login nuevo
- shell nuevo
- design system
- navegación por suites

### Fase 2
- dashboard ejecutivo
- CRM prospectos
- inventario conectado a pipeline

### Fase 3
- blackboard operativo
- gestoría y titulación renovadas
- seguimiento de requisiciones

### Fase 4
- permisos, auditoría y seguridad moderna
- limpieza de dependencias legacy
- capa de reportes premium
